# Brandivibe Marketing Strategy

This folder is the **knowledge base** for Brandivibe's sales motion. The upcoming AI sales brain reads these documents when researching leads, drafting emails, and handling objections.

Every document is built using the [April Dunford](https://www.aprildunford.com/) positioning framework and the PMM playbook from the `marketing-strategy-pmm` skill.

## Contents

| Document | What it contains | Who reads it |
|---|---|---|
| [`icp.md`](./icp.md) | Ideal Customer Profile — firmographics, psychographics, trigger events, 8-point scoring checklist, CRM tagging schema | AI brain (scoring leads), me (qualifying calls) |
| [`positioning.md`](./positioning.md) | Competitive alternatives, unique attributes, unique value, best-fit customer, market category, final positioning statement | AI brain (drafting hooks), me (pitching) |
| [`messaging.md`](./messaging.md) | Value props, benefit hierarchy, feature → outcome table, persona-specific messaging (economic buyer, technical buyer, champion) | AI brain (cold email tone), me (discovery calls) |
| [`battlecards.md`](./battlecards.md) | 5 competitive battlecards (basement.studio, studio freight, Webflow, in-house hire, do nothing) with talk tracks | AI brain (objection handling), me (objection handling) |
| [`gtm.md`](./gtm.md) | Sales-led motion, pipeline math, outbound flywheel diagram, trigger-based email rotation, 90-day launch plan, success metrics | AI brain (sourcing + scheduling), me (strategy review) |

## Key facts

- **Target**: Seed to Series B founders in crypto / SaaS / fintech / AI / luxury brands
- **Price**: $35K–$90K per project
- **Timeline**: 6 weeks fixed
- **Deliverable**: Next.js 16 + WebGL brand site + public GitHub repo + optional museum poster
- **Capacity**: 2 projects per quarter (solo studio)
- **Pipeline target**: 200 touches per month to close 2 projects per quarter

## How the AI sales brain uses this folder

When the brain handles a prospect, it loads these docs into context and uses them as the canonical source of truth for:

1. **Scoring** — run the prospect against `icp.md` checklist
2. **Matching** — pick the closest Brandivibe demo per industry
3. **Drafting** — use `messaging.md` persona + `positioning.md` hooks
4. **Responding** — match objections to `battlecards.md` talk tracks
5. **Scheduling** — use `gtm.md` flywheel steps 5-9

## When to update

- **icp.md** — after every 10 closed deals (recalibrate tier definitions)
- **positioning.md** — quarterly (or after a new competitor emerges)
- **messaging.md** — monthly (A/B test winners become canon)
- **battlecards.md** — when a new objection pattern shows up 3+ times
- **gtm.md** — quarterly based on actual conversion numbers
